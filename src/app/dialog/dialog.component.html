Mining for a pattern

Step 1 Choosing a pattern to mine for:

You can select from a preset of predetermined pattern or you can create your custom pattern. A pattern will be in a form of a regular expression.
The set of predetermined patterns are:

  1. Pattern 1: 0M1

    2. 0.M.1 -> Event-0 followed by a numeric expression followed by Event-1

  2. Pattern 2: 0M1M2

    3. 0.M.1.M.2 -> Event-0 followed by a numeric expression followed by Event-1 followed by a numeric expression followed by Event-2

  3. Pattern 3: 0M1M2M3

    4. 0.M.1.M.2.M.3 -> Event-0 followed by a numeric expression followed by Event-1 followed by a numeric expression followed by Event-2 and lastly followed by a numeric expression and then the last Event-3

You can also create your custom pattern by selecting the custom option, and then inputting your own QRET template into the available field. The QRET should be in the form of any sequence of XY where:
A number X which represents an event X.
An alphabet Y which represents the quantizer for the events that it follows
The system will then attempt to mine for any event in the trace that matches these pattern.

Step 2 Choosing the alphabet length:

The event length will determine how long the event will be. What this means is how many possible events are there. This number is capped at 26 at the moment, as any number bigger than that will severely slow the system, as the calculations will grow exponentially

Step 3 Choosing the file to mine:

The file must be of type x. You should also only attempt to mine one file at a time
